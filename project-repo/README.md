# project-repo

Мини-ассистент для продюсера: принимает сырой текст и автоматически раскладывает его по проектам и блокам.

## Как использовать

1. Настрой проекты в `data/projects.json`:
   - `id` — короткий идентификатор
   - `name` — название проекта
   - `aliases` — ключевые слова для автодетекта
   - `default_block` — блок по умолчанию

2. Отправь заметку:

```bash
python3 scripts/project_assistant.py "NPO дедлайн по лендингу сдвинулся на 2 дня, риск по согласованию макета"
```

Или через stdin:

```bash
cat note.txt | python3 scripts/project_assistant.py
```

3. Скрипт допишет заметку в `projects/<project_id>/log.md`.

## Как проект определяется

Приоритет:
1. `--project <id|name>`
2. Тег в тексте: `#project: NPO` или `проект: NPO`
3. Совпадения по `aliases`
4. Первый проект в списке (fallback)

## Как блок определяется

Если в тексте есть ключи (`статус`, `задача`, `риск`, `дедлайн`, `идея`, `операционка`), ассистент выберет соответствующий блок.
Иначе — `default_block` проекта.
