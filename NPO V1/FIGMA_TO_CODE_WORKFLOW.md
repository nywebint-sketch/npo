# Figma -> Code Workflow

Этот процесс нужен, чтобы быстро и стабильно переносить правки из Figma в `index.html`.

## 1) Что делать в Figma

- Правь макет в одном frame для конкретного брейкпоинта (например, mobile 390px).
- Не меняй все сразу: делай правки блоками (header, events, artists и т.д.).
- Фиксируй значения:
  - width/height
  - spacing (margin/padding/gap)
  - font-size/line-height/weight
  - colors/border/radius

## 2) Что экспортировать из Figma

- Экспорт frame в PDF/PNG (как референс).
- При необходимости экспортируй assets (иконки/картинки).
- Если поменялись токены — обнови `figma-tokens.json`.

## 3) Как ставить задачу в Codex

Используй такой шаблон:

```text
Внесли правки в Figma.
Файл: /absolute/path/to/Frame.pdf
Внеси изменения в /absolute/path/to/index.html
Брейкпоинт: mobile (390px)
Меняем только блоки: header, events
```

Если нужен точный перенос, добавляй:

```text
Сделай пиксель-в-пиксель под макет. Не трогай desktop.
```

## 4) Правило внесения в код

- Desktop стили не трогаем, если задача только про mobile.
- Правки делаем через media-query соответствующего брейкпоинта.
- Если удален блок в макете — удаляем:
  - HTML
  - связанные CSS
  - связанный JS
- После правки проверяем:
  - нет ли "мертвых" селекторов/функций
  - нет ли ссылок на удаленные `id`

## 5) Быстрый QA чеклист

- Header:
  - логотип/текст выровнены
  - высота topbar как в макете
- Сетки:
  - число карточек в ряд совпадает
  - одинаковые gap/паддинги
- Типографика:
  - размер/интерлиньяж/вес совпадают
- Интерактив:
  - карточки открываются
  - hover/active не ломают layout
- Чистота:
  - нет лишних кнопок/подписей, если в макете их нет

## 6) Рекомендуемый цикл

1. Экспорт из Figma.
2. Правка в коде (1 блок за раз).
3. Визуальная сверка.
4. Следующий блок.

Так правки получаются предсказуемыми и без поломок соседних секций.
